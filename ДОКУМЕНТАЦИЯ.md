## 1. Роли и иерархия

Система построена на трёхуровневой иерархии:

```
Босс (Директор / Суперадмин)
    └── Менеджер (Руководитель продаж)
            └── Сотрудник (Медицинский представитель)
```

### Что видит каждая роль

| Возможность | Босс | Менеджер | Сотрудник |
|---|---|---|---|
| Все продажи | Да | Только своих подчинённых | Только свои |
| Все аптеки/врачи | Да | Только своих подчинённых | Только свои |
| Аналитика по всем | Да | По своим подчинённым | Только своя |
| Дело сотрудника | Да | Только своих | — |
| Установка плана | Да | Только своим | — |
| Установка % ЗП | Только босс | — | — |

**Важно:** врачи и аптеки, которые создал сотрудник, видны только ему и его руководству. Другие сотрудники их не видят.

### Как назначить менеджера сотруднику

1. Зайти в **Админ-панель** → **Пользователи**
2. Открыть карточку сотрудника
3. В поле **Руководитель** выбрать нужного менеджера
4. Сохранить

## 3. Модуль CRM

Меню: **CRM → Врачи** / **CRM → Аптеки**

### Врачи

- Каждый сотрудник ведёт своих врачей — они не видны коллегам
- При создании врача поле **Представитель** автоматически ставится на текущего пользователя
- Можно прикреплять фото, указывать специальность, место работы, контакты

### Аптеки

- Аналогично врачам — каждый сотрудник видит только свои аптеки
- У аптеки отображается текущий долг — он обновляется автоматически при продажах и платежах
- Можно указать контактное лицо, адрес, телефон — они попадут в накладную

### Визиты

- Сотрудник фиксирует факт визита к врачу или в аптеку
- Можно прикрепить фото, аудиозапись, указать GPS-координаты
- Все визиты видны руководству в разделе аналитики

## 4. Модуль Склад

Меню: **Склад → Товары** / **Склад → Партии** / **Склад → Движения**

### Товары

На странице товаров показывается сводная таблица:

- Один товар = одна строка, даже если он поступал несколькими партиями
- **Общее количество** — сумма по всем партиям
- **Цена** — диапазон (мин–макс закупочная цена по партиям)
- **Маржа** — разница между отпускной ценой и закупочной

Чтобы увидеть отдельные партии — нажать на стрелку **▶** слева от строки товара.

### Приём товара (партия)

1. **Склад → Добавить партию**
2. Выбрать товар, указать серию/партию, срок годности, количество, закупочную цену, склад
3. Сохранить — количество появится на складе

### Сроки годности

- Красный цвет — товар просрочен
- Оранжевый — истекает в течение 90 дней
- Зелёный — в порядке

## 5. Модуль Продажи

Меню: **Продажи → Новая продажа**

### Оформление продажи

1. Выбрать **аптеку**
2. Выбрать **склад** (фильтрует доступные товары)
3. Указать **дату**
4. Выбрать **юридическое лицо** (продавец — ИП, ОсОО и т.д.)
5. Добавить товарные позиции:
   - Нажать **+ Добавить строку**
   - Начать вводить название товара в поисковое поле
   - Выбрать нужную партию из выпадающего списка
   - Указать количество (цена подставляется автоматически)
6. Указать **оплачено сейчас** — остаток автоматически уйдёт в долг аптеки
7. Прикрепить чек или фото документа (необязательно)
8. Нажать **Оформить продажу**

### Статусы продажи

| Статус | Описание |
|---|---|
| В обработке | Только создана, товар ещё не списан со склада |
| Подтверждена | Товар списан, долг аптеки обновлён |
| Отгружена | Отмечена как физически отгруженная |
| Отменена | Аннулирована |

**Важно:** списание товара со склада происходит только при нажатии кнопки **Подтвердить продажу**.

## 6. Долги и платежи

Меню: **Продажи → Долги**

### Как формируется долг

При подтверждении продажи:
```
Долг аптеки += (Сумма продажи − Оплачено при получении)
```

### Как принять оплату

1. Открыть **Продажи → Долги**
2. Найти аптеку
3. Нажать **Принять оплату**
4. Указать сумму и дату
5. Можно прикрепить фото чека
6. Нажать **Сохранить** — долг автоматически уменьшится

## 7. Аналитика

Меню: **Аналитика**

### Что показывает дашборд

- Общий объём продаж и маржа за период
- График продаж по месяцам (факт vs план)
- Топ сотрудников и аптек
- Разбивка по статусам продаж

### Фильтры

- Быстрые кнопки: Неделя / Месяц / Квартал / Год
- Произвольный диапазон дат

### Аналитика продаж

Меню: **Продажи → Аналитика**

- График продаж, себестоимости и прибыли по месяцам
- Топ сотрудников по объёму
- Топ аптек по объёму
- Воронка по статусам

## 8. Дело сотрудника

Меню: **Аккаунты → Пользователи → [выбрать сотрудника]**

Доступно только боссу и менеджеру.

### Что показывает карточка сотрудника

- Объём продаж за выбранный период
- Количество визитов
- Количество активных аптек и врачей
- График продаж помесячно
- Выполнение плана

### Зарплата

Рассчитывается автоматически:
```
ЗП = Сумма продаж за период × % ЗП сотрудника
```

По умолчанию процент = 7%. Босс может изменить его в карточке сотрудника.

### План продаж

1. Открыть карточку сотрудника
2. Нажать кнопку **✏️** напротив нужного месяца
3. Ввести плановую сумму
4. Сохранить

## 9. Накладные

### Создание накладной

Накладная формируется автоматически при создании продажи.

1. Открыть нужную продажу
2. Нажать **Скачать накладную**
3. В открывшейся странице нажать **Печать / Сохранить PDF**
4. В диалоге браузера выбрать **Сохранить как PDF**

### Содержание накладной

- Реквизиты продавца (юридическое лицо)
- Реквизиты покупателя (аптека: адрес, телефон, контакт)
- Таблица товаров: название, партия, срок годности, количество, цена, сумма
- Итог прописью (на русском языке)
- Информация об оплате и остатке долга
- Поля для подписей

## 10. Настройка юридических лиц

Меню: **Админ-панель → Юридические лица**

Перед первой продажей необходимо добавить хотя бы одно юридическое лицо.

### Что заполнить

| Поле | Описание |
|---|---|
| Наименование | Полное название, например: ИП «Арапова А.Х.» |
| Тип | ИП / ОсОО / ОАО / ЗАО / Другое |
| Адрес | Юридический адрес |
| Банк | Название банка |
| Р/счёт | Расчётный счёт |
| БИК | БИК банка |
| ИНН | ИНН организации |
| Свидетельство | Номер и дата свидетельства |
| По умолчанию | Если включено — подставляется автоматически в новых продажах |

## 11. Администрирование

### Создание нового пользователя

1. **Админ-панель** → **Пользователи** → **Добавить**
2. Указать имя, фамилию, логин, пароль
3. Выбрать **Роль**:
   - `med_rep` — Медицинский представитель
   - `sales_manager` — Руководитель продаж
   - `director` — Директор
   - `superadmin` — Суперадминистратор
4. Если сотрудник — указать **Руководитель** (кому подчиняется)
5. При необходимости изменить **% от продаж (ЗП)**

### Запуск сервера

```bash
cd /путь/к/проекту
source venv/bin/activate
python manage.py runserver
```

### Применение миграций (после изменений в моделях)

```bash
python manage.py makemigrations
python manage.py migrate
```

### Резервная копия базы данных

```bash
cp db.sqlite3 db.sqlite3.backup
```
